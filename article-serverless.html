Building CHOMA SENSE: How We Engineered a Precision Agriculture Tool

By Koren • Dec 19, 2025 • 8 min read

Precision agriculture is no longer just a buzzword. We built CHOMA SENSE, a mobile solution that empowers farmers with real-time data on soil moisture and pH levels, from tech stack to field validation.

Introduction

Precision agriculture is no longer just a buzzword—it is a necessity. For our latest project, CHOMA SENSE, we set out to build a mobile solution that empowers farmers with real-time data on soil moisture and pH levels. But building an app that is both technically robust and user-friendly for farmers requires more than just code; it requires a carefully planned development cycle.

Here is an inside look at how we built CHOMA SENSE, from the tech stack to the field tests at Sultan Kudarat State University.

1. The Blueprint: Our Tech Stack

To ensure the app was responsive and capable of handling real-time data, we chose a modern, efficient technology stack.

The Framework

We used Flutter (paired with the Dart language). This allowed us to build a cross-platform app that looks and feels native on any device.

The Engine

Visual Studio Code served as our primary web editing tool.

The Brains

We integrated Firebase for our backend. Firebase doesn't just store data; it allows for real-time synchronization. This means when a sensor detects a change in soil pH, the farmer sees it on their dashboard instantly. It also handles our user authentication, ensuring data remains secure.

2. The Process: Design Meets Function

We followed a strict Software Development Life Cycle (SDLC). We didn't want to just build a "tool"; we wanted to build an experience.

Our focus wasn't immediately on the hardware sensors (though the system supports Bluetooth/Wi-Fi integration). Instead, we prioritized Software Integration and UX. We prototyped key features like data logging and chart rendering first. We knew that if the dashboard wasn't intuitive, it wouldn't be used.

3. Field Testing at SKSU

A theory is only as good as its practice. To validate CHOMA SENSE, we conducted a study at the Sultan Kudarat State University (SKSU) Administration Building in Tacurong.

We utilized purposive sampling to find the perfect testers—people who actually understand the nuances of the problem we are solving:

The Users

8 Agriculture students provided feedback on how the app impacts farming methods.

The Experts

3 IT personnel evaluated the system's technical stability and code integrity.

4. Measuring Success

We didn't just ask, "Do you like it?" We used a rigorous rating scale questionnaire to measure four key pillars:

Effectiveness

Does it do the job?

Convenience

Is it easy to access?

Functionality

Do all features work as intended?

Efficiency

Does it perform without lag or errors?

Using descriptive statistics, we analyzed the data to create a user satisfaction profile. This statistical treatment allowed us to move beyond guesswork and prove that CHOMA SENSE is a reliable, effective tool for modern farming.

Conclusion

CHOMA SENSE is the result of merging agricultural science with computer engineering. By rigorously testing with real students and experts, we've created a foundation for smarter, data-driven farming.
